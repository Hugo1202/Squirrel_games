Documentación Técnica

El proyecto Squirrel Games es un ejemplo avanzado de simulación de una organización ficticia donde se aplican conceptos clave de Programación Orientada a Objetos (POO), como la herencia, polimorfismo y encapsulación. Además, sigue una arquitectura modular distribuida en paquetes temáticos.

Estructura del Proyecto

El proyecto está organizado de la siguiente manera:

        Squirrel_games/
        └── juego/
            ├── .classpath
            ├── .gitignore
            ├── .project
            ├── pom.xml                <- Configuración principal de Maven
            ├── src/
            │   ├── main/
            │   │   ├── java/
            │   │   │   ├── excepciones/
            │   │   │   └── juego/
            │   │   └── resources/      <- (Si es necesario, aquí irían recursos adicionales)
            │   └── test/
            │       ├── java/
            │       │   └── testJuego/  <- Tests unitarios
            │       └── resources/
            └── target/                  <- Generado por Maven (compilados)

La estructura sigue el estándar de Maven, facilitando la gestión de dependencias, construcción y ejecución.

Las clases principales son:

  • Juego.java: Controlador principal del juego.
  • Main.java: Punto de entrada.
  • Organization.java: Gestión de la organización y miembros.
  • Participantes.java: Participantes del juego.
  • Persona.java: Clase base común.
  • PinkGuards.java: Clase abstracta para guardias.
  • Managers.java: Supervisores.
  • Soldiers.java: Soldados armados.
  • Workers.java: Trabajadores.
  • Prueba.java: Define las pruebas.
  • TipoPrueba.java: Enum de tipos de prueba.
  • WeaponType.java: Enum de armas.
  • Department.java: Enum de departamentos.

En el paquete excepciones, se gestionan los errores específicos de negocio:

  • IdRepetidoException: Para IDs duplicados.
  • InfiltradoNoEliminableException: Maneja infiltrados especiales.
  • InvalidSupervisorException: Error de supervisión.
  • OrganizationException: Excepción general.

El proyecto cuenta con pruebas unitarias en el paquete testJuego, que validan el comportamiento de las principales funcionalidades del juego.
