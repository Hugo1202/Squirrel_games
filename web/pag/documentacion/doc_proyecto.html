Documentación del Proyecto

Squirrel Games es un proyecto académico cuyo objetivo es simular la gestión de una edición completa de unos juegos ficticios, combinando elementos de estrategia, azar y gestión de recursos humanos. Está inspirado en el popular "Squid Game", pero adaptado a un contexto de desarrollo de software educativo.

Objetivos del proyecto

  • Poner en práctica la Programación Orientada a Objetos (POO).
  • Aplicar manejo avanzado de excepciones personalizadas.
  • Gestionar un proyecto con Maven.
  • Desarrollar pruebas unitarias con JUnit.
  • Documentar el proyecto con Javadoc.

Requisitos Funcionales

  • Unicidad de identificadores para participantes.
  • Jerarquía de supervisión estricta entre Pink Guards.
  • Prohibición de eliminación de infiltrados.
  • Simulación de pruebas con porcentaje de eliminados.
  • Asignación obligatoria de un Manager como responsable de cada prueba.

Arquitectura

El proyecto sigue la estructura estándar de Maven:

            Squirrel_games/
            └── src/
                ├── main/java/ (Código principal)
                ├── test/java/ (Pruebas unitarias)
                └── resources/ (Recursos adicionales)

El sistema combina lógica de negocio (clases de dominio como Organization, Participantes), con un motor de reglas para validaciones y un conjunto de pruebas automáticas para garantizar la estabilidad.

¿Cómo se diseñó?

  • Modelo de dominio claro.
  • Separación de responsabilidades.
  • Pruebas desde el inicio (Test Driven Development).
  • Código documentado con Javadoc.

En resumen, Squirrel Games es mucho más que un simple juego: es una demostración de cómo aplicar buenas prácticas en un proyecto realista dentro de un entorno académico.
