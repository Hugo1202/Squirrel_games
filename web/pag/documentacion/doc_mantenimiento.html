Documentación de Mantenimiento

Para asegurar que Squirrel Games se mantenga correctamente a lo largo del tiempo, es importante seguir estas recomendaciones:

Actualización del Entorno

  • Verificar regularmente que Maven esté actualizado.
  • Revisar las versiones de dependencias en el pom.xml.
  • Si se actualiza Java, asegurar compatibilidad con plugins de Maven.

Mantenimiento del Código

  • Añadir pruebas unitarias nuevas para cada funcionalidad añadida.
  • Ejecutar las pruebas completas antes de cualquier release.
  • Documentar cada cambio relevante con Javadoc y README actualizado.

Control de Versiones

  • Usar Git para registrar cada modificación.
  • Dividir el trabajo en ramas para nuevas funcionalidades.
  • Revisar los commits de cada colaborador antes de hacer merge.

Organización del Repositorio

  • src/ para el código fuente.
  • test/ para las pruebas.
  • docs/ para documentación adicional (diagramas, informes, etc.).
  • .gitignore bien configurado para evitar subir basura.

Mejoras Futuras

  • Automatizar generación de reportes (cobertura de tests).
  • Publicar documentación y resultados de cobertura en GitHub Pages.
  • Ampliar funcionalidad del juego con nuevas pruebas o roles.

Con esta política de mantenimiento, Squirrel Games se garantiza como un proyecto sólido, escalable y fácil de evolucionar.
